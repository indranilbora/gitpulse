# Security Policy

## Supported Versions

Security fixes are prioritized for the latest release on `master` and the most recent tagged release.

## Reporting a Vulnerability

Use GitHub's private vulnerability reporting for this repository:

- Open the repository on GitHub
- Go to `Security` -> `Advisories` -> `Report a vulnerability`

If private reporting is unavailable, open a regular issue only for non-sensitive security hardening questions. Do not post exploit details or secrets in public issues.

## Response Expectations

- Initial triage acknowledgement target: within 3 business days
- Mitigation/fix timeline: based on severity and reproducibility
- Coordinated disclosure: preferred after a fix or mitigation is available
